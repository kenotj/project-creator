Architecture Design

Tauri + React + shadcn/ui — does this structure look right?

Repo Structure
project-creator/
src/ ← new React frontend
components/
AppLayout.tsx — root layout, wires panels
TemplateList.tsx — compact sidebar
TemplateEditor.tsx — editor + toolbar
FolderTree.tsx — recursive tree w/ inline rename
GenerateDialog.tsx — modal: name + destination
lib/
models.ts — FolderNode, Template types
store.ts — TemplateStore (CRUD + JSON persistence)
validation.ts — validateName()
generate.ts — generateProject() via Tauri fs
App.tsx — state, keyboard shortcuts (Cmd/Ctrl+S)
main.tsx
src-tauri/ ← Tauri config + Rust boilerplate only
tauri.conf.json
Cargo.toml
src/main.rs — no custom commands needed
tests/ ← Python tests → Vitest
models.test.ts
store.test.ts
validation.test.ts
generate.test.ts
ProjectCreator.py — kept as reference (unchanged)
package.json, vite.config.ts, tsconfig.json
Tauri APIs Used (no Rust code needed)
@tauri-apps/plugin-fs
Read/write ~/.projectcreator/templates.json
mkdir for project generation
@tauri-apps/plugin-dialog
Native folder picker (Browse button)
Success confirmation
@tauri-apps/api/path
homeDir() to locate
~/.projectcreator/
shadcn/ui components
Button, Input, Dialog, ScrollArea,
Badge, Separator + custom FolderTree
State Management
• App.tsx holds top-level state: templates[], selectedId, isDirty
• TemplateEditor manages local tree editing state (pending changes)
• No external state library needed — React useState + prop callbacks
• Unsaved-changes guard preserved (prompt before switching/deleting)

Does this architecture look right? Let me know any changes before we move to the UI mockup.
